Purpose

The meter component provides consistent styling for bounded scalar values, such as progress, score or completion.

Example

70%
{% highlight html %} 70% {% endhighlight %}

Usage notes

  • Use the native <meter> element for known ranges with a minimum and maximum.
  • Keep the fallback .c-meter markup in sync with the numeric value.
  • Use <progress> instead when showing task progress.

Accessibility notes

  • Provide a visible label or another accessible name for the meter.
  • Expose the actual value with min, max and value attributes.
  • Do not depend on the bar length alone; include nearby text when the exact value matters.
